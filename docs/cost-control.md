# 成本控制措施

## 1. AWS免费层使用策略

### 1.1 计算资源 (EC2)
- **实例类型**: 使用 t3.micro (1 vCPU, 1GB RAM) - 免费使用12个月
- **使用时间**: 限制实例运行时间，仅在工作时间运行，非工作时间自动停止
- **自动扩展**: 启用按需自动扩展，根据负载调整实例数量

### 1.2 存储资源 (S3)
- **存储类型**: 使用标准存储（前5GB免费）
- **生命周期策略**: 
  - 30天后自动转换为智能分层存储
  - 90天后自动转换为Glacier存储
- **版本控制**: 禁用版本控制以减少存储成本
- **访问日志**: 仅在必要时启用访问日志

### 1.3 数据库资源 (RDS)
- **实例类型**: 使用 db.t3.micro (前12个月免费)
- **存储**: 限制在20GB以内
- **快照**: 定期清理不必要的快照

### 1.4 网络资源
- **CloudFront**: 仅用于高流量静态资源，控制使用量
- **Elastic IP**: 仅在实例运行时分配，避免空闲EIP费用

## 2. GCP免费层使用策略

### 2.1 计算资源
- **Compute Engine**: 使用 f1-micro (1 vCPU, 0.6GB RAM) - 免费使用
- **Cloud Run**: 使用无服务器选项，仅按请求收费，控制请求量
- **使用时间**: 非工作时间自动停止实例

### 2.2 存储资源
- **Cloud Storage**: 使用标准存储（前5GB免费）
- **生命周期策略**: 
  - 30天后自动转换为近线存储
  - 90天后自动转换为归档存储
- **版本控制**: 禁用版本控制

### 2.3 数据库资源
- **Cloud SQL**: 使用 db-f1-micro (前12个月免费)
- **存储**: 限制在10GB以内

### 2.4 网络资源
- **Cloud CDN**: 控制使用量，仅用于高流量资源
- **外部IP**: 仅在实例运行时分配

## 3. 自动成本监控和告警

### 3.1 AWS成本监控
- **Billing Dashboard**: 设置预算和成本告警
- **Cost Explorer**: 定期分析成本趋势
- **AWS Budgets**: 创建月度预算，超过80%时发送告警
- **CloudWatch**: 监控资源使用情况，设置阈值告警

### 3.2 GCP成本监控
- **Billing Account**: 设置预算和成本告警
- **Cost Management**: 使用成本分析工具分析支出
- **Budgets & Alerts**: 创建月度预算，设置多阶段告警（50%、80%、100%）
- **Monitoring**: 监控资源使用情况，设置阈值告警

## 4. 资源优化策略

### 4.1 计算资源优化
- **实例调整**: 根据实际负载调整实例类型
- **自动启停**: 使用Lambda（AWS）或Cloud Functions（GCP）自动启停实例
- **容器化**: 使用Docker和Kubernetes优化资源利用率

### 4.2 存储资源优化
- **数据压缩**: 对存储的数据进行压缩
- **数据清理**: 定期清理不必要的数据
- **重复数据删除**: 避免存储重复数据

### 4.3 网络资源优化
- **内容分发**: 合理使用CDN，减少源站流量
- **流量优化**: 使用GZIP压缩减少传输数据量
- **缓存策略**: 优化缓存配置，减少重复请求

### 4.4 数据库优化
- **查询优化**: 优化数据库查询，减少CPU和内存使用
- **索引优化**: 合理创建索引，提高查询效率
- **连接池**: 使用连接池减少数据库连接开销

## 5. 成本分析和报告

### 5.1 定期成本分析
- **月度报告**: 生成月度成本报告，分析各服务支出
- **趋势分析**: 分析成本趋势，预测未来支出
- **异常检测**: 检测异常支出，及时处理

### 5.2 成本优化建议
- **资源闲置**: 识别和清理闲置资源
- **服务替代**: 评估是否有更便宜的服务替代方案
- **使用模式**: 优化资源使用模式，减少峰值使用

## 7. 成本控制最佳实践

1. **最小权限原则**: 仅分配必要的资源和权限
2. **定期审计**: 每月审计资源使用情况
3. **资源标记**: 对所有资源进行标记，便于成本分析
4. **预算约束**: 严格执行预算约束，避免超支
5. **持续优化**: 持续优化资源使用，降低成本

- **跨区域复制**: 仅在必要时使用跨区域复制
