# CircuitGPT 升级迁移指南

## 📦 快速开始

### 1. 前端升级

```bash
cd frontend

# 安装新依赖（Next.js 15.4+, React 19+）
npm install

# 验证构建
npm run build

# 启动开发服务器
npm run dev
```

**重要变化：**
- Next.js 14 → 15.4
- React 18 → 19
- MUI 5 → 6
- commonjs → ES 模块

### 2. 后端配置

```bash
cd backend

# 生成安全的 SECRET_KEY
python -c 'import secrets; print(secrets.token_urlsafe(32))'

# 复制示例配置
cp .env.example .env

# 编辑 .env 文件，填入生成的 SECRET_KEY
# 必须至少32个字符！
```

**.env 示例：**
```env
SECRET_KEY=你生成的随机密钥字符串-至少32字符
DEBUG=true
DB_URL=sqlite:///./app.db
REDIS_URL=redis://localhost:6379
AI_SERVICE_URL=http://localhost:8001
EDA_SERVICE_URL=http://localhost:8002
CORS_ORIGINS=http://localhost:3000
```

详细配置说明见：[backend/CONFIG_SETUP.md](../backend/CONFIG_SETUP.md)

### 3. 启动服务

```bash
# 在项目根目录

# 启动后端
./scripts/run-backend.sh

# 启动前端（新终端）
./scripts/run-frontend.sh

# 启动AI服务（新终端）
./scripts/run-ai-service.sh

# 启动EDA服务（新终端）
./scripts/run-eda-service.sh
```

## ⚠️ 可能遇到的问题

### 问题1: npm install 失败

**原因：** Node.js 版本过低或 package-lock.json 冲突

**解决：**
```bash
# 检查 Node.js 版本（需要 18+）
node --version

# 清理并重新安装
rm -rf node_modules package-lock.json
npm install
```

### 问题2: SECRET_KEY 验证失败

**错误信息：**
```
SECRET_KEY is insecure! It must be:
  - At least 32 characters long
  - Not a common/default value
```

**解决：**
```bash
# 生成新的安全密钥
python -c 'import secrets; print(secrets.token_urlsafe(32))'

# 或使用 OpenSSL
openssl rand -base64 32
```

将生成的密钥填入 `.env` 文件的 `SECRET_KEY`。

### 问题3: React 19 兼容性警告

**现象：** 控制台出现一些弃用警告

**原因：** 某些第三方库可能还未完全兼容 React 19

**解决：** 
- 这些警告不影响功能
- 等待库作者更新
- 或暂时忽略这些警告

### 问题4: WebSocket 连接失败

**现象：** 页面显示 "连接模式: 轮询"

**原因：** Socket.IO 服务未启动或端口被占用

**解决：**
1. 检查后端服务是否正常运行
2. 检查防火墙设置
3. 验证 `WEBSOCKET_URL` 配置正确
4. 轮询模式作为降级方案也能正常工作
