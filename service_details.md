# AI驱动自动化电路设计系统 - 服务详细职责和接口

## 1. 前端层

### 1.1 Next.js 14+ 前端应用
- **职责**：提供用户界面，处理用户输入，展示设计结果
- **核心功能**：
  - 自然语言输入界面
  - 设计过程实时展示
  - 设计结果可视化（原理图、PCB、BOM）
  - 用户交互和修改功能
- **接口**：
  - HTTP REST API：与FastAPI后端通信
  - WebSocket：通过Socket.io接收实时设计进度

### 1.2 Material-UI 组件库
- **职责**：提供UI组件，实现响应式设计
- **核心功能**：
  - 表单组件
  - 可视化组件
  - 响应式布局
- **接口**：
  - 与Next.js前端集成

### 1.3 SVG.js 可视化
- **职责**：渲染和编辑SVG格式的电路原理图
- **核心功能**：
  - SVG图形渲染
  - 交互式编辑
  - 缩放和平移功能
- **接口**：
  - 与Next.js前端集成

### 1.4 Socket.io-client 实时通信
- **职责**：建立WebSocket连接，接收实时设计进度
- **核心功能**：
  - 实时消息接收
  - 连接管理
- **接口**：
  - 与Socket.io服务器通信

## 2. API网关层

### 2.1 FastAPI 后端API服务
- **职责**：系统核心API网关，处理所有客户端请求
- **核心功能**：
  - 请求路由
  - 身份验证
  - 数据验证
  - 错误处理
- **主要接口**：
  - `POST /api/design`：提交电路设计请求
  - `GET /api/design/{id}`：获取设计结果
  - `PUT /api/design/{id}`：修改设计参数
  - `DELETE /api/design/{id}`：删除设计

### 2.2 JWT 认证服务
- **职责**：用户身份验证和授权
- **核心功能**：
  - 令牌生成
  - 令牌验证
  - 用户会话管理
- **接口**：
  - `POST /api/auth/login`：用户登录
  - `POST /api/auth/register`：用户注册
  - `GET /api/auth/me`：获取当前用户信息

### 2.3 Socket.io 服务器
- **职责**：提供实时通信服务
- **核心功能**：
  - 设计进度推送
  - 用户通知
  - 实时协作支持
- **接口**：
  - WebSocket连接端点
  - 事件处理（design_progress, design_complete等）

## 3. AI服务层

### 3.1 Flask AI服务
- **职责**：提供AI驱动的电路设计能力
- **核心功能**：
  - 自然语言解析
  - 电路设计生成
  - 设计优化建议
- **接口**：
  - `POST /api/ai/parse`：解析自然语言请求
  - `POST /api/ai/generate`：生成电路设计

### 3.2 CircuitBERT 电路设计模型
- **职责**：AI电路设计核心模型
- **核心功能**：
  - 电路拓扑生成
  - 组件选择推荐
  - 设计规则检查
- **接口**：
  - 与Flask AI服务集成的Python API

### 3.3 NLP 自然语言处理
- **职责**：解析用户的自然语言输入
- **核心功能**：
  - 意图识别
  - 实体提取
  - 需求分析
- **接口**：
  - 与Flask AI服务集成的Python API

## 4. EDA工具链层

### 4.1 SKiDL 原理图生成
- **职责**：生成电路原理图
- **核心功能**：
  - 电路组件布局
  - 连接关系生成
  - 原理图验证
- **输入**：
  - 电路需求（组件列表、连接关系）
- **输出**：
  - SKiDL格式原理图
  - SVG可视化文件

### 4.2 PySpice 电路模拟
- **职责**：模拟电路性能
- **核心功能**：
  - 电路行为模拟
  - 性能参数计算
  - 故障检测
- **输入**：
  - SKiDL格式原理图
  - 模拟参数
- **输出**：
  - 模拟结果数据
  - 波形图

### 4.3 KiCad PCB自动布线
- **职责**：生成PCB布局
- **核心功能**：
  - 组件布局规划
  - 自动布线
  - DRC（设计规则检查）
- **输入**：
  - 原理图
  - PCB设计规则
- **输出**：
  - KiCad PCB文件
  - 3D视图

### 4.4 BOM 生成服务
- **职责**：生成物料清单
- **核心功能**：
  - 组件列表提取
  - 供应商信息匹配
  - 成本估算
- **输入**：
  - 原理图
  - PCB设计
- **输出**：
  - BOM表格（CSV/Excel）

## 5. 异步处理层

### 5.1 Celery 任务队列
- **职责**：处理异步任务
- **核心功能**：
  - 任务调度
  - 任务执行
  - 结果管理
- **主要任务**：
  - 电路设计生成
  - 模拟计算
  - PCB布局
  - BOM生成

### 5.2 Redis 消息代理
- **职责**：为Celery提供消息传递
- **核心功能**：
  - 任务队列存储
  - 结果存储
  - 缓存

## 6. 存储层

### 6.1 SQLite 数据库
- **职责**：存储系统数据
- **核心功能**：
  - 用户信息存储
  - 设计项目管理
  - 设计参数存储
- **主要表结构**：
  - users：用户信息
  - designs：设计项目
  - design_parameters：设计参数
  - design_results：设计结果

### 6.2 文件系统存储
- **职责**：存储设计文件
- **核心功能**：
  - 原理图文件存储
  - 模拟结果存储
  - PCB文件存储
  - BOM文件存储
- **存储结构**：
  - `/designs/{design_id}/schema/`：原理图文件
  - `/designs/{design_id}/simulation/`：模拟结果
  - `/designs/{design_id}/pcb/`：PCB文件
  - `/designs/{design_id}/bom/`：BOM文件
